Dify는 오픈소스 LLM 앱 개발 플랫폼입니다. 직관적인 인터페이스로 AI 워크플로우, RAG 파이프라인, 에이전트 기능, 모델 관리, 관측 기능([Opik](https://www.comet.com/docs/opik/integrations/dify), [Langfuse](https://docs.langfuse.com), [Arize Phoenix](https://docs.arize.com/phoenix) 포함) 등을 제공하여 프로토타입에서 프로덕션까지 빠르게 전환할 수 있습니다. 핵심 기능은 다음과 같습니다:

## Quick start

> Dify를 설치하기 전에 시스템이 다음 최소 요구 사항을 충족하는지 확인하세요:
>
> - CPU >= 2 Core
> - RAM >= 4 GiB

<br/>

Dify 서버를 시작하는 가장 쉬운 방법은 [Docker Compose](docker/docker-compose.yaml)를 사용하는 것입니다. 다음 명령어를 실행하기 전에 [Docker](https://docs.docker.com/get-docker/)와 [Docker Compose](https://docs.docker.com/compose/install/)가 설치되어 있어야 합니다:

```bash
cd dify
cd docker
cp .env.example .env
docker compose up -d
```

실행 후 브라우저에서 [http://localhost/install](http://localhost/install)로 Dify 대시보드에 접속하여 초기화 과정을 시작할 수 있습니다.

#### 도움 요청

설치 중 문제가 발생하면 [FAQ](https://docs.dify.ai/getting-started/install-self-hosted/faqs)를 참조하세요. 여전히 해결되지 않으면 [커뮤니티에 문의](#community--contact)하세요.

> Dify에 기여하거나 추가 개발을 원하시면 [소스 코드 배포 가이드](https://docs.dify.ai/getting-started/install-self-hosted/local-source-code)를 참조하세요

## 핵심 기능

**1. 워크플로우**:
비주얼 캔버스에서 강력한 AI 워크플로우를 구축하고 테스트합니다. 아래의 모든 기능과 그 이상을 활용할 수 있습니다.

**2. 포괄적인 모델 지원**:
수십 개의 추론 프로바이더 및 자체 호스팅 솔루션에서 제공하는 수백 개의 상용/오픈소스 LLM과 원활하게 통합됩니다. GPT, Mistral, Llama3 및 모든 OpenAI API 호환 모델을 지원합니다. 지원되는 모델 프로바이더의 전체 목록은 [여기](https://docs.dify.ai/getting-started/readme/model-providers)에서 확인할 수 있습니다.

![providers-v5](https://github.com/langgenius/dify/assets/13230914/5a17bdbe-097a-4100-8363-40255b70f6e3)

**3. 프롬프트 IDE**:
프롬프트 작성, 모델 성능 비교, 챗 기반 앱에 TTS 등 추가 기능을 제공하는 직관적인 인터페이스입니다.

**4. RAG 파이프라인**:
문서 수집부터 검색까지 포괄적인 RAG 기능을 제공합니다. PDF, PPT 및 기타 일반적인 문서 형식에서 텍스트 추출을 기본 지원합니다.

**5. 에이전트 기능**:
LLM Function Calling 또는 ReAct 기반으로 에이전트를 정의하고, 사전 구축되거나 커스텀 도구를 추가할 수 있습니다. Dify는 Google Search, DALL·E, Stable Diffusion, WolframAlpha 등 50개 이상의 내장 도구를 제공합니다.

**6. LLMOps**:
시간 경과에 따른 애플리케이션 로그와 성능을 모니터링하고 분석합니다. 프로덕션 데이터와 어노테이션을 기반으로 프롬프트, 데이터셋, 모델을 지속적으로 개선할 수 있습니다.

**7. Backend-as-a-Service**:
Dify의 모든 기능에 해당하는 API를 제공하여, 자체 비즈니스 로직에 Dify를 손쉽게 통합할 수 있습니다.

## Dify 사용법

- **Cloud <br/>**
  설정 없이 누구나 사용할 수 있는 [Dify Cloud](https://dify.ai) 서비스를 호스팅합니다. 자체 배포 버전의 모든 기능을 제공하며, 샌드박스 플랜에 200회의 무료 GPT-4 호출이 포함되어 있습니다.

- **Self-hosting Dify Community Edition<br/>**
  이 [스타터 가이드](#quick-start)로 자체 환경에서 Dify를 빠르게 실행할 수 있습니다.
  자세한 참고 자료와 심층적인 지침은 [공식 문서](https://docs.dify.ai)를 확인하세요.

- **Dify for enterprise / organizations<br/>**
  엔터프라이즈 중심의 추가 기능을 제공합니다. 엔터프라이즈 요구 사항을 논의하려면 [이메일로 문의](mailto:business@dify.ai?subject=%5BGitHub%5DBusiness%20License%20Inquiry)해 주세요. <br/>

  > AWS를 사용하는 스타트업과 소기업은 [AWS Marketplace의 Dify Premium](https://aws.amazon.com/marketplace/pp/prodview-t22mebxzwjhu6)을 확인하고 원클릭으로 자체 AWS VPC에 배포하세요. 커스텀 로고와 브랜딩으로 앱을 생성할 수 있는 저렴한 AMI 오퍼링입니다.

## 고급 설정

### 커스텀 설정

설정을 커스터마이즈하려면 `docker/.env`를 편집하세요. 필수 시작 기본값은 [`docker/.env.example`](docker/.env.example)에 있으며, 선택적 고급 변수는 `docker/envs/` 아래에 테마별로 분리되어 있습니다. 변경 후 `docker` 디렉토리에서 `docker compose up -d`를 다시 실행하세요. 사용 가능한 환경 변수의 전체 목록은 [여기](https://docs.dify.ai/getting-started/install-self-hosted/environments)에서 확인할 수 있습니다.

### Grafana 메트릭 모니터링

Dify의 PostgreSQL 데이터베이스를 데이터 소스로 사용하여 Grafana에 대시보드를 임포트하고, 앱, 테넌트, 메시지 등의 세부 메트릭을 모니터링할 수 있습니다.

- [Grafana Dashboard by @bowenliang123](https://github.com/bowenliang123/dify-grafana-dashboard)

### Kubernetes 배포

고가용성 설정이 필요한 경우, 커뮤니티에서 기여한 [Helm Charts](https://helm.sh/)와 YAML 파일을 사용하여 Kubernetes에 Dify를 배포할 수 있습니다.

- [Helm Chart by @LeoQuote](https://github.com/douban/charts/tree/master/charts/dify)
- [Helm Chart by @BorisPolonsky](https://github.com/BorisPolonsky/dify-helm)
- [Helm Chart by @magicsong](https://github.com/magicsong/ai-charts)
- [YAML file by @Winson-030](https://github.com/Winson-030/dify-kubernetes)
- [YAML file by @wyy-holding](https://github.com/wyy-holding/dify-k8s)
- [🚀 NEW! YAML files (Supports Dify v1.6.0) by @Zhoneym](https://github.com/Zhoneym/DifyAI-Kubernetes)

#### Terraform 배포

[terraform](https://www.terraform.io/)을 사용하여 클라우드 플랫폼에 원클릭 배포

##### Azure Global

- [Azure Terraform by @nikawang](https://github.com/nikawang/dify-azure-terraform)

##### Google Cloud

- [Google Cloud Terraform by @sotazum](https://github.com/DeNA/dify-google-cloud-terraform)

#### AWS CDK 배포

[CDK](https://aws.amazon.com/cdk/)를 사용하여 AWS에 Dify 배포

##### AWS

- [AWS CDK by @KevinZhao (EKS based)](https://github.com/aws-samples/solution-for-deploying-dify-on-aws)
- [AWS CDK by @tmokmss (ECS based)](https://github.com/aws-samples/dify-self-hosted-on-aws)

#### Alibaba Cloud Computing Nest 사용

[Alibaba Cloud Computing Nest](https://computenest.console.aliyun.com/service/instance/create/default?type=user&ServiceName=Dify%E7%A4%BE%E5%8C%BA%E7%89%88)로 알리바바 클라우드에 빠르게 배포

#### Alibaba Cloud Data Management 사용

[Alibaba Cloud Data Management](https://www.alibabacloud.com/help/en/dms/dify-in-invitational-preview/)로 알리바바 클라우드에 원클릭 배포

#### Azure DevOps Pipeline로 AKS 배포
